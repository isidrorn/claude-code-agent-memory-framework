For the senior-engineering-skill-pack repo: **commit and push to `main` freely.** Do **not** create
branches yet — branch only when we need to validate or test different approaches side by side.

**Why:** early-stage solo curation repo; the overhead of branch/PR isn't worth it until there are
competing approaches to compare.

**How to apply:** stage → commit → push to `main` without asking each time. Still write clear commit
messages. Revisit (start branching) once the user says we're validating alternatives.
